# Security Policy

## Supported Versions

The following versions of Kuberna Labs are currently being supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

Kuberna Labs takes security very seriously, particularly regarding our smart contracts, agent marketplace, and backend services. 

If you discover a vulnerability, please report it to us immediately rather than filing a public issue. We ask that you do not share details of the vulnerability publicly or with third parties until it has been verified and patched by our team.

Please report security issues to:
**security@kuberna.africa**

### What to include in your report
- A detailed description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact (e.g., unauthorized access, funds at risk, privilege escalation).
- Any proof of concept (PoC) code or specific files/lines involved.

### Our Response Process
1. We will acknowledge receipt of your vulnerability report within 48 hours.
2. We will investigate the issue and communicate our findings to you.
3. If confirmed, we will provide a timeline for a patch.
4. After the patch is deployed and verified, we will publicly disclose the vulnerability (crediting you if desired).

### Bug Bounty
For severe vulnerabilities concerning smart contract funds, escaping TEE sandboxes, or critical backend database exploits, we operate an active bug bounty program. Reach out to the email above to coordinate bounty assessments before publishing.
